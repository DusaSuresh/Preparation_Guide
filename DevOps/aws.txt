AWS Overview and Details for Technical Architect Interview

1. Introduction to AWS

What is AWS?
Amazon Web Services (AWS) is a comprehensive cloud computing platform provided by Amazon. It offers a broad range of services, including computing power, storage, databases, and machine learning, allowing businesses to scale and manage their infrastructure efficiently.

Why Use AWS?

Scalability: Easily scale resources up or down based on demand.
Flexibility: Choose from a wide range of services and configurations.
Global Reach: Deploy applications in multiple regions and availability zones.
Security: Comprehensive security features and compliance certifications.
2. Core AWS Services

Compute

Amazon EC2 (Elastic Compute Cloud): Scalable virtual servers for various applications. Supports multiple instance types for different workloads.

Example:

bash
Copy code
# Launch an EC2 instance using the AWS CLI
aws ec2 run-instances --image-id ami-0abcdef1234567890 --count 1 --instance-type t2.micro --key-name MyKeyPair
Amazon ECS (Elastic Container Service): Managed service for running Docker containers. Supports both EC2 and Fargate launch types.

Example:

bash
Copy code
# Create an ECS cluster
aws ecs create-cluster --cluster-name my-cluster
Amazon EKS (Elastic Kubernetes Service): Managed Kubernetes service for running Kubernetes clusters on AWS.

Example:

bash
Copy code
# Create an EKS cluster
aws eks create-cluster --name my-cluster --role-arn <role-arn> --resources-vpc-config subnetIds=<subnet-ids>,securityGroupIds=<security-group-ids>
AWS Lambda: Serverless computing service that lets you run code in response to events without provisioning servers.

Example:

bash
Copy code
# Create a Lambda function
aws lambda create-function --function-name my-function --runtime nodejs14.x --role <role-arn> --handler index.handler --zip-file fileb://function.zip
Storage

Amazon S3 (Simple Storage Service): Scalable object storage for data backup, archival, and analytics.

Example:

bash
Copy code
# Upload a file to S3
aws s3 cp myfile.txt s3://my-bucket/
Amazon EBS (Elastic Block Store): Block storage volumes for use with EC2 instances.

Example:

bash
Copy code
# Create an EBS volume
aws ec2 create-volume --size 8 --availability-zone us-west-2a --volume-type gp2
Amazon EFS (Elastic File System): Scalable file storage for use with AWS Cloud services and on-premises resources.

Example:

bash
Copy code
# Create an EFS file system
aws efs create-file-system --creation-token my-token
Databases

Amazon RDS (Relational Database Service): Managed relational database service supporting various database engines (MySQL, PostgreSQL, MariaDB, Oracle, SQL Server).

Example:

bash
Copy code
# Create an RDS instance
aws rds create-db-instance --db-instance-identifier mydbinstance --db-instance-class db.t2.micro --engine mysql --master-username admin --master-user-password password
Amazon DynamoDB: Fully managed NoSQL database service with support for key-value and document data models.

Example:

bash
Copy code
# Create a DynamoDB table
aws dynamodb create-table --table-name my-table --attribute-definitions AttributeName=Id,AttributeType=S --key-schema AttributeName=Id,KeyType=HASH --provisioned-throughput ReadCapacityUnits=5,WriteCapacityUnits=5
Amazon Redshift: Managed data warehouse service for large-scale data storage and analytics.

Example:

bash
Copy code
# Create a Redshift cluster
aws redshift create-cluster --cluster-id my-cluster --node-type dc2.large --master-username admin --master-user-password password --db-name mydb

11. Security Best Practices

Network Security

VPC Security Groups: Define rules to control inbound and outbound traffic to your instances.
VPC Network ACLs: Apply rules to control traffic at the subnet level.
Data Protection

Encryption: Use AWS KMS to encrypt data at rest and in transit.
Backup: Regularly back up data using services like Amazon RDS automated backups and Amazon S3 versioning.
Access Management

IAM Roles and Policies: Apply least privilege principle to limit access.
Multi-Factor Authentication (MFA): Enable MFA for AWS accounts to enhance security.
12. AWS in Production

High Availability

Multi-AZ Deployments: Distribute resources across multiple Availability Zones to ensure fault tolerance.
Auto Scaling: Automatically scale resources based on demand.
Disaster Recovery

Backup and Restore: Regularly back up data and test restore procedures.
Cross-Region Replication: Replicate data across regions for additional redundancy.
